Memory Taxonomy

The useful split is not just vector vs graph. A real system needs to separate what is remembered, how it is stored, and who is allowed to reuse it.

Multi-Agent Memory

Multi-agent memory fails when every agent writes to the same pile. The safer design is private scratch memory, promoted shared facts, and clear deletion rules.

Design Stance

The most useful first system is boring in the right places: inspectable writes, scoped reads, and simple retrieval before graph complexity.

Start with Memory Inbox

Agents propose candidate memories. Users or policies approve promotion into durable memory.

Separate scopes

User, project, agent run, and shared team memories should not live in one undifferentiated bucket.

Keep deletion real

Every derived index should be rebuildable from approved source memories so delete/export stay credible.
